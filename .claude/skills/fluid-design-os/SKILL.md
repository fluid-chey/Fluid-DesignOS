# Fluid Design OS

Start and manage the Fluid Design OS canvas -- a visual workspace for reviewing, annotating, and iterating on generated marketing assets.

## Behavior

Parse `$ARGUMENTS` to determine action:

### Stop (argument contains "stop")

Run the stop script and confirm:

```bash
bash canvas/scripts/stop.sh
```

Print: "Canvas stopped."

### Start (default, no arguments or anything other than "stop")

1. **Check dependencies:** If `canvas/node_modules` does not exist, install them:
   ```bash
   cd canvas && npm install
   ```

2. **Launch the canvas dev server:**
   ```bash
   bash canvas/scripts/start.sh
   ```

3. **Print instructions:**
   ```
   Fluid Design OS canvas is running.

     Canvas:  http://localhost:5174

   Quick start:
     1. Open http://localhost:5174 in your browser
     2. Generate assets: /fluid-social "your topic" --variations 3
     3. View variations in the canvas grid
     4. Click on a variation to place pin annotations
     5. Mark a winner, add notes, then iterate

   MCP agent bridge:
     The MCP agent bridge (fluid-canvas) is auto-managed by Claude Code via
     .mcp.json. When Claude Code needs to use MCP tools (push_asset,
     read_annotations, read_history, etc.), it spawns the MCP server
     automatically. No manual MCP process management is needed.

   Stop: /fluid-design-OS stop
   ```

## Architecture Notes

- **start.sh** manages only the Vite dev server process. It writes a PID file
  to `canvas/.pid` for lifecycle tracking.
- **stop.sh** reads the PID file and kills the dev server process.
- The **MCP server** is a separate stdio process managed by Claude Code's MCP
  infrastructure. It is configured in `.mcp.json` at the project root (created
  in Plan 02). Claude Code reads `.mcp.json` and spawns the MCP server process
  automatically when it needs to use MCP tools. The start/stop scripts do NOT
  manage the MCP server.
- Assets generated by `/fluid-social --variations N` are written to
  `.fluid/working/{sessionId}/` directories, which the canvas discovers and
  displays automatically via file watching.

## Anti-Patterns

- **Do NOT** start or stop the MCP server from these scripts. Claude Code
  handles MCP lifecycle via .mcp.json.
- **Do NOT** run `npm run dev` directly -- always use `canvas/scripts/start.sh`
  so the PID file is managed correctly.
