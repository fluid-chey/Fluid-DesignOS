Create `archetypes/SPEC.md` — the formal, authoritative archetype format specification. This is the primary deliverable of Phase 18 and the document Phase 19 implementers will follow. Structure:

**Title:** "Archetype Format Specification v1.0"

**Section 1: Overview**
- An archetype is a brandless structural layout pattern for social media content
- Each archetype = one directory under `archetypes/` containing 3 files: index.html, schema.json, README.md
- Archetypes define CONTENT LAYOUT ONLY — decorative elements are brand-defined

**Section 2: File Structure**
- `index.html` — Brandless structural skeleton. Renderable in a browser. Uses neutral placeholder styling (grayscale, sans-serif, no brand URLs). Contains `<!-- SLOT: field-name -->` comments before each content element.
- `schema.json` — SlotSchema definition matching `canvas/src/lib/slot-schema.ts` interface exactly. Contains: archetypeId (string, metadata-only), width (number), height (number), fields (SlotField[]), brush (always null).
- `README.md` — Purpose, structural pattern description, content type this archetype fits, when to use / not use, which components it composes.

**Section 3: HTML Conventions**
- DOCTYPE html required, lang="en"
- Meta charset UTF-8
- All CSS in a single `<style>` block in `<head>` — NO inline styles, NO external stylesheets
- `body` dimensions match target platform (1080x1080 for Instagram, 1200x627 for LinkedIn)
- `overflow: hidden` on body
- `font-family: sans-serif` (brand font injected at generation)
- `background: #111` (neutral dark — brand background applied at generation)
- All content elements use absolute positioning with explicit px values
- Each content element has a unique CSS class matching its schema.json `sel` value
- `<!-- SLOT: {class-name} -->` HTML comment before each content element
- A `.decorative-zone` div (position: absolute, full canvas, pointer-events: none, z-index: 0) at the top of body for brand decorative injection
- Placeholder text in all content elements (e.g., "94%", "YOUR HEADLINE HERE", "Body copy placeholder") — NOT empty divs
- Colors: white text (#ffffff) with reduced opacity variants (rgba(255,255,255,0.45) for body, rgba(255,255,255,0.5) for labels) — neutral scheme replaced by brand at generation

**Section 4: schema.json Rules**
- MUST match the SlotSchema TypeScript interface from slot-schema.ts
- `archetypeId` field (string) — metadata identifier, NOT `templateId` (templateId keys into TEMPLATE_SCHEMAS and would cause incorrect resolution in resolveSlotSchemaForIteration)
- `width` and `height` — platform dimensions in pixels
- `fields` array — ordered list of SlotField objects. Each field:
  - `type`: "text" or "image" (never "divider" for single-frame archetypes)
  - `sel`: CSS selector matching the HTML class (e.g., ".headline", ".stat-number", ".photo img")
  - `label`: Human-readable label for editor sidebar
  - `mode`: For text fields — "text" (single-line feel), "pre" (multi-line preserving whitespace), "br" (line breaks)
  - `rows`: For text fields — hint for textarea height (1 for short labels, 2 for headlines, 4-5 for body copy)
  - `dims`: For image fields — display dimension hint (e.g., "353 x 439px")
  - `frameSel`: For image fields — optional explicit frame selector. If omitted and sel ends with ` img`, imageLayoutSel() derives the parent (`.photo img` -> `.photo`)
- `brush`: MUST be `null` — decorative brush selectors are brand-defined, merged by pipeline
- `brushLabel`: omit
- `brushAdditional`: omit
- `carouselCount`: omit for single-frame archetypes
- Full JSON example (stat-hero-single):
  ```json
  {
    "archetypeId": "stat-hero-single",
    "width": 1080,
    "height": 1080,
    "fields": [
      { "type": "text", "sel": ".category span", "label": "Side Label",    "mode": "text", "rows": 1 },
      { "type": "text", "sel": ".context-label", "label": "Context Label", "mode": "text", "rows": 1 },
      { "type": "text", "sel": ".stat-number",   "label": "Stat Value",    "mode": "text", "rows": 1 },
      { "type": "text", "sel": ".headline",      "label": "Headline",      "mode": "text", "rows": 2 },
      { "type": "text", "sel": ".body-copy",     "label": "Body Copy",     "mode": "pre",  "rows": 4 }
    ],
    "brush": null
  }
  ```

**Section 5: Content/Decorative Split**
- CONTENT (archetype-defined): text blocks, image blocks, stat values, headlines, body copy, attribution, CTAs
- DECORATIVE (brand-defined): brushstrokes, circle sketches, background textures, gradient overlays, brand watermarks
- At generation time, the pipeline: (1) reads archetype index.html as structural skeleton, (2) reads archetype schema.json for content fields, (3) injects brand decorative elements into .decorative-zone, (4) merges brand brush/brushAdditional fields into the final iteration SlotSchema, (5) applies brand fonts, colors, and styling to content elements
- The archetype skeleton is renderable WITHOUT brand assets — it shows layout structure with placeholder text and neutral styling

**Section 6: CSS Selector Convention**
- Use flat CSS class selectors (`.headline`, `.stat-number`, `.body-copy`) not nested or complex selectors
- For image fields: `.{container} img` pattern (e.g., `.photo img`) so imageLayoutSel() derives `.photo` as the draggable frame
- Use `<!-- SLOT: {class} -->` comments matching the selector class name
- Do NOT use `data-slide` prefixes (that is for carousel templates only)
- Do NOT use BEM, utility classes, or CSS frameworks — plain semantic class names only

**Section 7: Compatibility Checklist**
Checklist that every archetype must pass before shipping:
- [ ] schema.json parses as valid JSON
- [ ] schema.json has archetypeId, width, height, fields array, brush: null
- [ ] Every field.sel in schema.json has a matching CSS class in index.html
- [ ] Every `<!-- SLOT: -->` comment in HTML has a matching field in schema.json
- [ ] index.html renders in a browser at the target dimensions showing placeholder content
- [ ] collectTransformTargets(schema) produces a valid TransformTarget[] with correct kinds
- [ ] No templateId field in schema.json
- [ ] No brand asset URLs in index.html
- [ ] No inline styles in index.html (all styling in style block)
- [ ] All content elements have placeholder text (not empty)

**Section 8: Anti-Patterns**
List from RESEARCH.md anti-patterns section:
- No brand asset URLs in archetype HTML
- No templateId in schema.json
- No decorative CSS rules (brushstrokes, circles, textures) in archetype stylesheets
- No empty structural markup — always use placeholder content
- No components that are too atomic (single text label is not a component)

**Section 9: Platform Dimensions Reference**
| Platform | Width | Height | Phase |
|----------|-------|--------|-------|
| Instagram Square | 1080 | 1080 | 19 |
| LinkedIn Landscape | 1200 | 627 | 21 |
| One-Pager (US Letter) | 612 | 792 | 21 |
