  <name>Task 1: Refactor /api/generate for multi-asset campaign creation</name>
  <files>canvas/src/server/watcher.ts</files>
  <action>
    Refactor the `/api/generate` POST handler in watcher.ts. This is the largest change in Phase 8. Read the full existing handler carefully before modifying.

    **1. Replace single activeChild with parallel tracking:**
    ```typescript
    // Remove: let activeChild: ChildProcess | null = null;
    // Add:
    const activeChildren: Map<string, ChildProcess> = new Map(); // keyed by assetId
    let activeCampaignGeneration: string | null = null; // campaign-level lock
    ```

    **2. Channel parsing from prompt:**
    Create a `parseChannelHints(prompt: string)` function that detects keywords:
    - "just linkedin" / "linkedin only" / "only linkedin" -> ['linkedin']
    - "just instagram" / "instagram only" / "only instagram" -> ['instagram']
    - "one-pager only" / "just a one-pager" -> ['one-pager']
    - No hints -> default full spread
    Case-insensitive regex matching. Return `{ channels: string[], assetCounts: Record<string, number> }` where defaults are `{ instagram: 3, linkedin: 3, 'one-pager': 1 }`.

    **3. Build default asset list:**
    ```typescript
    function buildAssetList(channels: Record<string, number>): Array<{ title: string; assetType: string; frameCount: number }> {
      const assets: Array<{ title: string; assetType: string; frameCount: number }> = [];
      for (const [type, count] of Object.entries(channels)) {
        for (let i = 1; i <= count; i++) {
          assets.push({ title: `${type} ${i}`, assetType: type, frameCount: 1 });
        }
      }
      return assets;
    }
    ```

    **4. Pre-create full structure BEFORE spawning:**
    In the handler:
    - Call `parseChannelHints(prompt)` to determine what to create
    - If `req.body.existingCampaignId` is provided, skip campaign creation and add assets to existing campaign
    - Otherwise: `createCampaignWithAssets({ title: first30CharsOfPrompt, channels }, assetList)`
    - For each asset: `createFrame({ assetId: asset.id, frameIndex: 0 })`
    - For each frame: pre-allocate iterationId with `nanoid()`, create canonical htmlPath `campaigns/${campaignId}/${assetId}/${frameId}/${iterationId}.html`, `fs.mkdirSync(path.dirname(absPath), { recursive: true })`, then `createIteration({ frameId, iterationIndex: 0, htmlPath, source: 'ai', generationStatus: 'pending' })`
    - Stream SSE session event with campaignId immediately: `res.write(data: { type: 'session', campaignId, ... })`

    **5. Spawn N parallel subagents:**
    For each asset, spawn a separate `claude` subprocess:
    - Build per-asset prompt: include the user's prompt, the asset type, target dimensions (`instagram: 1080x1080, linkedin: 1200x627, one-pager: 816x1056`), the exact output path to write HTML to, and instruction to call `PATCH /api/assets/{assetId}` with a descriptive title after generating.
    - Use `spawn('claude', ['-p', assetPrompt, '--allowedTools', ...])` same as existing pattern.
    - Track in `activeChildren.set(assetId, child)`.
    - Multiplex stdout onto the SSE connection with `assetId` field in each event frame.
    - On `child.close`: call `updateIterationGenerationStatus(iterationId, 'complete')`, delete from activeChildren map.

    **6. Wait for ALL children before sending done event:**
    Track a counter or use Promise.all pattern. Only send `event: done` when all N children have closed. This prevents the auto-navigate race condition described in RESEARCH.md Pitfall 5.

    **7. Campaign-level lock:**
    If `activeCampaignGeneration !== null` when a new full campaign generation is requested, return 409. Clear `activeCampaignGeneration` when all children close. Iterations on existing assets (iterate mode) should NOT be blocked by this lock.

    **8. Stop creating .fluid/working/ for new generations:**
    For the new campaign-centric path, do NOT create session directories under `.fluid/working/`. The canonical path is `.fluid/campaigns/...` only. Keep the old code path for iterate-on-existing flows that still reference working dirs (if any), but guard it behind a check.

    **IMPORTANT:** Do NOT break the existing iterate flow. When `req.body.iterateMode === true`, the existing single-asset iteration path should still work — it creates a new Iteration under an existing Frame.
  </action>
